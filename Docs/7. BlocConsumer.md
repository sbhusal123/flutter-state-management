# Bloc Consumer

- `BlocConsumer` is used when we need to listen for a state change as well as rebuild a UI. 

- Unlik `BlocListner` is used when we only need to listen for state change.

`BlocConsumer` should only be used when it is necessary to both rebuild UI and execute other reactions to state changes in the bloc.


```dart
BlocConsumer<BlocA, BlocAState>(
    builder: (context, state) => {
        return <Widget>
    },
    listner: (context, state) {
        // do some operations, navigate, snackbar
    },
    buildWhen: (prevState, nextState) => {
        return <bool>
    },
    listenWhen: (prevState, nextState) => {
        return <bool>
    }
)
```


## Usage:

```dart
            BlocConsumer<CounterBloc, CounterState>(
              listenWhen: (previous, current) => true, // true by default
              buildWhen: (previous, current) => true, // true by default
              listener: (context, state) {
                if(state.count == 3){
                  ScaffoldMessenger.of(context).showSnackBar(
                    SnackBar(content: const Text("Value is 3"))
                  );
                }
              },
              builder: (context, state) {
                return Text(
                  '${state.count}',
                  style: Theme.of(context).textTheme.headlineMedium,
                );
              },
            ),
```
