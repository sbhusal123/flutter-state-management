# Handling Multiple Events

To handle multiple events we need to define a **sealed** class for events instead of abstract class.

And extend each event with the **sealed** class as below:

**bloc/counter_event.bloc**

```dart
sealed class CounterEvent {}

class CounterIncreamentEvent extends CounterEvent {}

class CounterDecreamentEvent extends CounterEvent {}
```

Now we can handle multiple events in bloc class as below:

**bloc/counter_bloc.bloc**

```dart

import 'package:flutter_bloc/flutter_bloc.dart';
import 'package:src/bloc/counter_event.dart';
import 'package:src/bloc/counter_state.dart';

Class CounterBloc extends Bloc<CounterEvent, CounterState> {

    CounterBloc(): super(const CounterState(count: 0)){
        on<CounterIncreamentEvent>( 
            (event, emit) => emit(CounterState(state.count+1))
        );

        on<CounterDecreamentEvent>( 
            (event, emit) => emit(CounterState(state.count-1))
        );
    }
}
```

## Sealed Class

- class modifier used to create a known, restricted hierarchy of classes. 

- allows you to define a finite set of subtypes, which gives the Dart compiler complete awareness of every possible variation of that class.

- All subclasses of a sealed class must be defined in the exact same file (or library) as the sealed class itself.

**Example:**

```dart
sealed class Result {
  const Result();
}

class Success extends Result {
  final String data;
  const Success(this.data);
}

class Failure extends Result {
  final String error;
  const Failure(this.error);
}

// Compiler knows these are the ONLY possible types.
```
